# 🎙️ Gpb.VoiceTranscription

**Gpb.VoiceTranscription** — это десктопное WPF-приложение для транскрибации аудиофайлов в текст с использованием модели Whisper. Приложение поддерживает автоматическую конвертацию аудио в совместимый формат, предоставляет прогресс обработки и позволяет экспортировать результат в различных форматах.

## ✨ Возможности

- **Транскрибация аудио** — преобразование речи в текст с помощью модели Whisper (OpenAI)
- **Запись аудио из приложения** — запись голоса напрямую через интерфейс программы
  - **Запись с микрофона** — выбор любого доступного микрофона системы
  - **Запись системного звука (Loopback)** — захват аудио с наушников/динамиков (Wasapi Loopback)
  - **Кнопки управления** — «⏺️ Запись» и «⏹️ Стоп» для контроля процесса
  - **Автоматическая транскрибация** — после остановки записи файл сразу обрабатывается
- **Автоматическая конвертация** — приведение аудио к формату, совместимому с Whisper (16kHz, Mono, 16-bit PCM)
- **Drag & Drop** — удобная загрузка файлов перетаскиванием
- **Прогресс обработки** — отображение статуса и процента выполнения в реальном времени
- **Экспорт результатов** — сохранение в форматах Markdown, JSON, SRT (субтитры), Plain Text
- **Таймкоды** — каждый сегмент текста сопровождается временными метками
- **Отмена операции** — возможность прервать процесс транскрибации
- **Современный UI** — тёмная тема в стиле Visual Studio

## 📦 Установка и запуск

### Предварительные требования

- **ОС**: Windows 10/11
- **.NET SDK**: версия 10.0 или выше
- **IDE**: Visual Studio 2022 / JetBrains Rider / VS Code (опционально)

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd Gpb.VoiceTranscription
```

### Шаг 2: Восстановление зависимостей

```bash
dotnet restore
```

### Шаг 3: Сборка проекта

```bash
dotnet build --configuration Release
```

### Шаг 4: Запуск приложения

```bash
dotnet run
```

Или откройте проект в Visual Studio и нажмите `F5`.

> **Примечание**: При первом запуске модель Whisper (`ggml-base.bin`) будет автоматически загружена в папку `Assets/Models/`.

## 🚀 Использование

### Базовый сценарий (транскрибация файла)

1. **Запустите приложение**
2. **Выберите WAV-файл**:
   - Нажмите кнопку **«🔍 Обзор...»** или
   - Перетащите файл в область Drag & Drop
3. **(Опционально)** Установите флажок **«⚙️ Автоматически конвертировать аудио»**, если ваш файл не в формате 16kHz Mono WAV
4. **Нажмите «▶️ Начать»** для запуска транскрибации
5. **Дождитесь завершения** — прогресс отображается в реальном времени
6. **Сохраните результат** кнопкой **«💾 Сохранить .md»**

### Запись аудио из приложения

1. **Выберите источник звука** в выпадающем списке:
   - 🎤 Микрофон (любой доступный в системе)
   - 🔊 Системный звук (Loopback) — для записи звука из наушников/динамиков
2. **Нажмите «⏺️ Запись»** — начнётся запись аудио
3. **Говорите в микрофон** или воспроизведите звук, который нужно записать
4. **Нажмите «⏹️ Стоп»** — запись остановится
5. **Автоматическая транскрибация** — после остановки записи файл будет сразу обработан
6. **Сохраните результат** при необходимости

### Поддерживаемые действия

| Действие | Описание |
|----------|----------|
| **Выбор файла** | Открытие диалога выбора WAV-файла |
| **Drag & Drop** | Перетаскивание файла из проводника |
| **Запись (микрофон)** | Запись голоса с выбранного микрофона |
| **Запись (Loopback)** | Запись системного звука (наушники/динамики) |
| **Старт транскрибации** | Запуск процесса транскрибации |
| **Стоп записи** | Остановка записи и начало транскрибации |
| **Отмена** | Прерывание текущей операции |
| **Сохранение** | Экспорт результата в Markdown |

## 🧩 Описание компонентов

### Models

#### `TranscriptionResult`
Класс, представляющий полный результат транскрибации:
- **Свойства**:
  - `SourceFilePath` — путь к исходному файлу
  - `DetectedLanguage` — распознанный язык
  - `ModelName` — использованная модель Whisper
  - `StartedAt` / `CompletedAt` — временные метки
  - `ProcessingTimeMs` — длительность обработки
  - `Segments` — коллекция сегментов
- **Методы**:
  - `ToMarkdown()` — экспорт в Markdown
  - `ToSrt()` — экспорт в формат субтитров SRT
  - `ToJson()` — экспорт в JSON
  - `SaveToFile()` — сохранение в файл

#### `TranscriptionSegment`
Отдельный сегмент транскрибации:
- **Свойства**:
  - `Start` / `End` — временные интервалы (TimeSpan)
  - `Text` — распознанный текст
  - `Confidence` — уверенность модели (опционально)
  - `SpeakerId` — ID спикера (для будущего расширения)
- **Методы**:
  - `ToMarkdownLine()` — форматирование для Markdown
  - `ToSrtTimestamp()` — форматирование для SRT

#### `ExportFormat`
Перечисление поддерживаемых форматов:
- `Markdown`
- `PlainText`
- `Json`
- `Srt`

---

### ViewModels

#### `MainViewModel`
Основная ViewModel, управляющая логикой приложения:
- **Observable свойства**:
  - `StatusMessage` — текущий статус
  - `ProgressValue` — процент выполнения (0–100)
  - `IsProcessing` — флаг активности
  - `SelectedFilePath` — путь к выбранному файлу
  - `TranscriptionResult` — результат транскрибации
  - `AutoConvertAudio` — флаг автоконвертации
  - `IsRecording` — флаг активной записи
  - `SelectedDeviceIndex` — индекс выбранного устройства записи
  - `AvailableDevices` — список доступных устройств (микрофоны + Loopback)
- **Команды**:
  - `SelectFileCommand` — выбор файла
  - `StartTranscriptionCommand` — запуск транскрибации
  - `CancelCommand` — отмена операции
  - `SaveResultCommand` — сохранение результата
  - `StartRecordingCommand` — начало записи аудио
  - `StopRecordingCommand` — остановка записи
- **Методы**:
  - `InitializeAsync()` — инициализация сервиса Whisper
  - `StartTranscriptionAsync()` — основная логика обработки
  - `Dispose()` — освобождение ресурсов

---

### Services

#### `WhisperTranscriptionService`
Сервис для работы с моделью Whisper:
- **Методы**:
  - `InitializeAsync()` — загрузка/инициализация модели
  - `TranscribeAsync()` — асинхронная транскрибация файла
  - `DownloadModelAsync()` — загрузка модели при отсутствии
- **События**:
  - `ProgressChanged` — текстовый статус
  - `ProgressPercentChanged` — процент выполнения

#### `AudioConverter`
Утилитарный класс для конвертации аудио:
- **Методы**:
  - `IsWhisperCompatible()` — проверка соответствия требованиям
  - `ConvertToWhisperFormatAsync()` — конвертация в 16kHz Mono 16-bit PCM
- **Особенности**:
  - Сведение стерео в моно
  - Ресемплинг до 16kHz (WdlResamplingSampleProvider)
  - Конвертация IEEE Float → PCM 16-bit

#### `AudioRecordingService`
Сервис для записи аудио с микрофона или системного звука:
- **Методы**:
  - `StartRecordingAsync(deviceIndex, useLoopback)` — начало записи
  - `StopRecordingAsync()` — остановка и возврат пути к файлу
  - `GetAvailableDevices()` — получение списка доступных устройств
- **События**:
  - `RecordingStateChanged` — изменение состояния записи
  - `ErrorOccurred` — ошибка при записи
- **Особенности**:
  - Поддержка записи с микрофона (WaveInEvent)
  - Поддержка записи системного звука (WasapiCapture Loopback)
  - Автоматическая запись в формате 16kHz Mono 16-bit PCM для Whisper

## 🎵 Требования к аудио

Модель Whisper требует следующие параметры аудио:

| Параметр | Значение |
|----------|----------|
| **Формат** | WAV |
| **Частота дискретизации** | 16 kHz |
| **Каналы** | Mono (1 канал) |
| **Разрядность** | 16-bit PCM |
| **Кодирование** | PCM (WaveFormatEncoding.Pcm) |

Если ваш файл не соответствует этим требованиям, включите опцию **«Автоматически конвертировать аудио»** — приложение выполнит конвертацию автоматически.

## ⚙ Конфигурация и настройки

### Путь к модели
По умолчанию модель сохраняется в:
```
./Assets/Models/ggml-base.bin
```

Изменить путь можно в `MainWindow.xaml.cs`:
```csharp
var modelPath = Path.Combine(
    Environment.CurrentDirectory,
    "Assets", "Models", "ggml-base.bin");
```

### Выбор модели Whisper
В текущей версии используется модель **base**. Для изменения отредактируйте вызов в `MainViewModel.cs`:
```csharp
await _transcriptionService.TranscribeAsync(
    processPath,
    modelName: "small", // tiny/base/small/medium/large
    _cts.Token);
```

### Язык распознавания
По умолчанию установлен `auto` (автоматическое определение). Можно указать конкретный язык в конструкторе `WhisperTranscriptionService`:
```csharp
new WhisperTranscriptionService(modelPath, language: "ru")
```

## 🔧 Расширение функциональности

### Добавление новых форматов экспорта
1. Добавьте значение в enum `ExportFormat`
2. Реализуйте метод в `TranscriptionResult.cs`
3. Обновите `switch` в методе `SaveToFile()`

### Поддержка других моделей
Добавьте загрузку дополнительных моделей в `WhisperTranscriptionService`:
```csharp
await DownloadModelAsync(fileName, GgmlType.Small);
```

### Интеграция diarization (разделение спикеров)
Используйте поле `SpeakerId` в `TranscriptionSegment` для хранения идентификатора спикера после интеграции с библиотеками diarization.
